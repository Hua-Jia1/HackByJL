## 第 0 部分：选工具的 3 个原则

> 工具不是越多越好，**够用 + 熟练 = 高效**。

### 原则 1：一个工具解决一类问题

| 场景 | 工具（只选一个） |
|------|-----------------|
| 终端 | **MobaXterm**（Windows）/ iTerm2（macOS） |
| 监控 | **Zabbix**（传统）/ Prometheus + Grafana（云原生） |
| 备份配置 | **Oxidized**（首选）/ Netmiko 脚本（自写） |
| 资产管理 | **NetBox**（首选）/ Excel 凑合 |
| 文档 | **语雀 / 飞书文档 / 钉钉文档**（国内） |
| 密码 | **Bitwarden**（首选） |

### 原则 2：先开源后商业

| 免费 / 开源 | 对应商业版 |
|------------|----------|
| Zabbix | SolarWinds / PRTG |
| LibreNMS | SolarWinds NPM |
| Oxidized | SolarWinds NCM |
| NetBox | NetBox Labs 商业版 |
| Wireshark | SolarWinds Response Time Viewer |
| Grafana | Datadog / New Relic |

> **商业版不是不能买**，但先用开源跑通，**真有需求再上商业**。

### 原则 3：先自动化后智能化

```
手动 → 脚本自动化 → 平台化 → AI 增强
```

| 阶段    | 做法                             |
| ----- | ------------------------------ |
| 手动    | 人工 SSH 改配置                     |
| 脚本    | Netmiko 批量改                    |
| 平台    | Ansible / SaltStack / Oxidized |
| AI 增强 | ChatGPT 排错 + 自动生成报告            |

## 第 1 部分：日常高频（每天必用）

### 1.1 MobaXterm（Windows 神器）

**官网**：https://mobaxterm.mobatek.net

**为什么是它**：
- SSH + Telnet + RDP + VNC + SFTP + FTP + 串口 + X11 全在一个窗口
- 内置 X 服务器（Linux GUI 直接转发）
- 多标签 / 多窗口 / 会话保存
- 内置密码管理（不要用，建议外挂 Bitwarden）
- 文件拖拽上传下载

**实战用法**：

```
场景 1：批量登录多台设备
- Session -> New Session -> SSH
- 填 IP / 用户名 / 密码
- 保存到 "Ruijie" / "Huawei" / "H3C" 分组
- 下次直接点分组展开

场景 2：串口救机
- Session -> New Session -> Serial
- 选 COM 口 / 波特率 9600
- 设备宕机也能进

场景 3：SFTP 传文件
- 连上 SSH 后左侧自动开 SFTP
- 拖拽文件上传
- 直接看服务器上的文件（图片也能预览）

场景 4：宏命令（多台设备同样操作）
- Tools -> Macro
- 录命令 -> 在多窗口同时回放
```

**替代品**：Xshell + Xftp（付费但稳）/ Termius（跨平台）/ Tabby（开源新秀）

### 1.2 Wireshark（抓包之王）

**官网**：https://www.wireshark.org

**实战用法**：

```
场景 1：业务不通，怀疑策略
- 抓包看 SYN 是否到达对端
- 看 RST 谁发的（多半是防火墙）

场景 2：网站慢，定位哪一跳
- TCP Stream Graphs -> Round Trip Time
- 看哪一段延迟高

场景 3：DHCP 拿不到 IP
- 过滤: bootp
- 看 Discover / Offer / Request / Ack 四步

场景 4：DNS 解析异常
- 过滤: dns
- 看响应是否正常
```

**过滤表达式速查**（**运维要会写**）：

| 表达式 | 含义 |
|--------|------|
| `ip.addr == 192.168.1.1` | 这个 IP 的所有包 |
| `tcp.port == 80` | 80 端口的 TCP |
| `tcp.flags.syn == 1` | SYN 包 |
| `tcp.flags.reset == 1` | RST 包（重置） |
| `dns` | 所有 DNS |
| `icmp` | ping / traceroute |
| `tcp.analysis.retransmission` | 重传包 |
| `http.response.code == 500` | HTTP 500 |
| `ssl.handshake` | TLS 握手 |
| `frame.time_delta > 1` | 包间隔超 1 秒 |

**推荐**：**VS Code**

**运维要装的插件**：

| 插件 | 用途 |

| Remote - SSH | 远程直接编辑服务器配置 |
| YAML | 写 Ansible / K8s |
| Markdown All in One | 写文档 |
| Mermaid | 画架构图 |
| Draw.io Integration | 画拓扑 |
| GitLens | 看 Git 历史 |

**实战用法**：

### 1.4 Bitwarden（密码管理）

**官网**：https://bitwarden.com

**为什么必须用**：
- 网络设备账号几百个，**不统一管理就是定时炸弹**
- 离职交接只需要交接 1 个 master 密码
- 多人共享 vault（团队版）

**实战用法**：
- 给每台设备建一条记录
- 字段：设备名 / 厂商 / 型号 / IP / Console 账号密码 / SSH 账号密码 / Web 控制台
- 配合 MobaXterm："Tools -> KeePass Integration"（Bitwarden 兼容 KeePass 格式）

## 第 2 部分：批量与自动化（解放双手）

### 2.1 Netmiko（Python 库，批量改配置）

**官网**：https://github.com/ktbyers/netmiko

**安装**：`pip install netmiko`

**实战模板**（你已经有这个了，再加个常用场景）：

```python
# 场景：批量给 10 台交换机改 NTP 服务器
from netmiko import ConnectHandler
from concurrent.futures import ThreadPoolExecutor

devices = [
    {"name": "sw-1", "host": "10.1.1.1", "device_type": "huawei", "username": "admin", "password": "xxx"},
    {"name": "sw-2", "host": "10.1.1.2", "device_type": "huawei", "username": "admin", "password": "xxx"},
    # ...
]

commands = [
    "system-view",
    "ntp server 10.99.1.1",
    "ntp server 10.99.1.2",
    "return",
    "save",
]

def config_device(device):
    try:
        with ConnectHandler(**device) as conn:
            output = conn.send_config_set(commands)
            print(f"✅ {device['name']} OK")
            return True
    except Exception as e:
        print(f"❌ {device['name']} {e}")
        return False

# 5 个并发（不要太多，触发安全告警）
with ThreadPoolExecutor(max_workers=5) as pool:
    results = list(pool.map(config_device, devices))

print(f"成功: {sum(results)}/{len(devices)}")
```

**用 Netmiko 干的活**：
- 批量改 NTP
- 批量加 VLAN
- 批量改 ACL
- 批量加 SNMP 配置
- 批量重启
- 批量采集配置

### 2.2 Ansible（自动化运维事实标准）

**官网**：https://www.ansible.com

**运维角度的两个核心模块**：
- `ios_command` / `huawei_command` / `h3c_command`：发命令
- `ios_config` / `huawei_config`：改配置

**实战：批量配 NTP**（Ansible 写法）：

```yaml
# ntp.yml
- name: 批量配置 NTP
  hosts: switches
  gather_facts: no

  tasks:
    - name: 配置 NTP 服务器
      huawei_config:
        lines:
          - ntp server 10.99.1.1
          - ntp server 10.99.1.2
        save: yes
```

**为什么比 Netmiko 好**：
- 幂等（重复执行结果一样）
- 有 Playbook（流程化）
- 有 Inventory（设备清单）
- 有 Roles（复用）

### 2.3 Nornir（Netmiko 升级版，Python 原生）

**官网**：https://github.com/nornir-automation/nornir

**为什么选它**：
- 不用写 YAML
- Python 原生（脚本里直接调）
- 并发内置
- Netmiko、NAPALM、Scrapli 都能接

```python
from nornir import InitNornir
from nornir.core.task import Task, Result
from nornir_netmiko.tasks import netmiko_send_config

nr = InitNornir(config_file="config.yaml")

def config_ntp(task: Task) -> Result:
    task.run(
        task=netmiko_send_config,
        config_commands=[
            "system-view",
            "ntp server 10.99.1.1",
            "save",
        ]
    )

result = nr.run(task=config_ntp)
print(result)
```

### 2.4 Paramiko（SSH 底层库，Netmiko 也是基于它）

**适合**：自己写工具的"造轮子"场景
**不推荐**：日常用 Netmiko / Nornir 就够了

### 2.5 SSH 跳板机工具

| 工具 | 特点 |
|------|------|
| **Jumpserver** | 国内最流行的开源堡垒机 |
| Teleport | 开源，更现代 |
| WindTerm | 终端 + SFTP + 跳板 |
| ProxyJump（OpenSSH） | ssh config 里 `ProxyJump` 一行搞定 |

**最简的跳板方案**（无需堡垒机）：

```bash
# ~/.ssh/config
Host jump
  HostName 10.99.1.1
  User admin

Host switch-*
  ProxyJump jump
  User admin

# 用法
ssh switch-10-1-1-1
```

## 第 3 部分：诊断与排错（应急救命）

### 3.1 MTR（看路径上的丢包）

**为什么用它**：traceroute 只能看路径，**MTR 还能看到每一跳的丢包率**。

**安装**：
- Windows: `winget install mtr` 或下载 BestTrace
- macOS: `brew install mtr`
- Linux: `apt install mtr` / `yum install mtr`

**实战**：

```bash
mtr -r -c 100 8.8.8.8
# -r: report 模式
# -c 100: 发 100 个包统计
```

**输出解读**：
```
HOST: Loss%  Snt  Last  Avg  Best  Wrst
 1. 10.1.1.1     0%   100  0.5  0.5  0.6  0.7
 2. 10.99.1.1    0%   100  1.2  1.5  1.0  2.3
 3. ...          30%  100  10   15   8    50  ← 30% 丢包，是这一跳的锅
 4. ...
```

**结论**：丢包发生在哪一跳，**就是哪一跳的设备/链路问题**。

### 3.2 iperf3（测真实带宽）

**安装**：
```bash
apt install iperf3   # Linux
brew install iperf3  # macOS
# Windows: 下载 iperf3.exe
```

**实战**：

```bash
# 服务端（一台机器）
iperf3 -s

# 客户端（另一台机器）
iperf3 -c <server-ip>        # TCP 测试
iperf3 -c <server-ip> -u     # UDP 测试
iperf3 -c <server-ip> -P 4   # 4 并发连接
```

**常用场景**：
- 验证 ISP 给的带宽（运营商说 100M，实测 80M = 亏了）
- 验证内网链路（千兆线实测只有 200M = 哪里有问题）
- 验证 VPN 隧道（IPSec 性能）

### 3.3 Netcat / ncat（端口测试）

**实战**：

```bash
# 测试端口是否通
nc -zv 192.168.1.1 80
# 22 (ssh) open
# 80 (http) open
# 23 (telnet) Connection refused

# 听端口（模拟服务）
nc -l 8080

# 测带宽
nc -l 9999 > /dev/null
# 另一台
dd if=/dev/zero bs=1M count=1000 | nc <server> 9999
```

### 3.4 dig / nslookup（DNS 排查）

**实战**：

```bash
# 查 A 记录
dig google.com
dig +short google.com

# 查 MX
dig google.com MX

# 查 NS
dig google.com NS

# 指定 DNS 服务器查
dig @8.8.8.8 google.com

# 反向解析
dig -x 8.8.8.8

# 跟踪解析过程
dig +trace google.com

# 测试 DNS 响应时间
dig google.com | grep "Query time"
```

**DNS 故障排查清单**：
1. 是否能 ping DNS 服务器 → 物理通不通
2. 是否能 TCP/UDP 53 端口 → 防火墙通不通
3. 解析是否返回正确 IP → DNS 服务器是否对
4. 响应时间 → 是否慢
5. 是否被劫持（返回错误 IP）→ 安全问题

### 3.5 curl / wget（HTTP 测试）

**实战**：

```bash
# 看 HTTP 头
curl -I https://www.baidu.com

# 看完整响应（调试用）
curl -v https://www.baidu.com 2>&1 | head -30

# 测 HTTPS 证书
curl -vI https://www.baidu.com 2>&1 | grep -i "expire\|subject"

# 看返回时间
curl -o /dev/null -s -w "time_total: %{time_total}\n" https://www.baidu.com

# 模拟 POST
curl -X POST -H "Content-Type: application/json" -d '{"name":"test"}' https://api.example.com

# 走代理
curl --proxy http://proxy:8080 https://www.baidu.com

# 跟随重定向
curl -L https://bit.ly/some-link
```

### 3.6 tcpdump（服务器抓包）

**安装**：`apt install tcpdump`

**实战**：

```bash
# 看某网卡所有流量
tcpdump -i eth0

# 看某 IP 的包
tcpdump -i eth0 host 192.168.1.1

# 看某端口
tcpdump -i eth0 port 80

# 看某协议
tcpdump -i eth0 icmp

# 组合
tcpdump -i eth0 src 192.168.1.1 and dst port 80

# 写到文件（用 Wireshark 打开）
tcpdump -i eth0 -w capture.pcap

# 不解析名字（更详细）
tcpdump -i eth0 -nn

# 只抓 N 个包
tcpdump -i eth0 -c 100
```

### 3.7 Nmap（端口扫描 / 资产发现）

**安装**：`apt install nmap` / `brew install nmap`

**实战**：

```bash
# 扫主机存活
nmap -sn 192.168.1.0/24

# 扫端口
nmap 192.168.1.1

# 扫端口 + 服务 + 版本
nmap -sV 192.168.1.1

# 操作系统识别
nmap -O 192.168.1.1

# 全端口扫描（慢）
nmap -p- 192.168.1.1

# 扫网段
nmap 192.168.1.0/24

# 扫漏洞（用脚本）
nmap --script vuln 192.168.1.1
```

> ⚠️ **生产环境慎用**！可能触发安全告警。先和领导/客户打招呼。

### 3.8 hping3（高级 ping，能发 SYN）

**实战**：

```bash
# 发 SYN
hping3 -S -p 80 192.168.1.1

# 看防火墙对 SYN 的反应
# -S = SYN, -A = ACK, -F = FIN
```

**常用场景**：
- 防火墙对 SYN 放行但对 ACK 拦了 → 配置问题
- 测试端口是否被过滤

### 3.9 SmokePing（持续网络质量监控）

**官网**：https://oss.oetiker.ch/smokeping/

**干嘛的**：连续 ping 几十个目标，**画成图表**看网络质量的长期趋势。

**场景**：
- 监控到 8.8.8.8 / 114.114.114.114 的延迟变化
- 监控到关键业务的可达率
- 出问题后能看出"从哪天开始抖"

## 第 4 部分：监控与告警（自建 vs SaaS）

### 4.1 自建路线

#### Zabbix（综合监控之王）

**官网**：https://www.zabbix.com

**优势**：
- SNMP / IPMI / JMX / HTTP / SSH / Agent 全支持
- 模板丰富（思科/华为/华三/锐捷都有官方/民间模板）
- 告警灵活（电话/短信/微信/钉钉/邮件）
- 大屏内置

**实战模板**（以交换机为例）：

```yaml
# 监控项（Item）
CPU 使用率：
  Key: 1.3.6.1.4.1.2011.5.25.31.1.1.1.1.5.0
  # 这是华为 OID，CPU 5 秒平均

内存使用率：
  Key: 1.3.6.1.4.1.2011.5.25.31.1.1.1.1.7.0

接口入流量：
  Key: ifInOctets[Gi0/0/1]
  
接口出流量：
  Key: ifOutOctets[Gi0/0/1]

接口错包：
  Key: ifInErrors[Gi0/0/1]

# 触发器（Trigger）
CPU 高：
  {host:cpu.last()} > 80

接口流量超 80%：
  {host:ifInOctets[Gi0/0/1].last()} / 100000000 > 800  # 单位换算

设备失联：
  {host:icmpping.last(5)} = 0
```

#### Prometheus + Grafana（云原生首选）

**官网**：
- Prometheus: https://prometheus.io
- Grafana: https://grafana.com

**优势**：
- 时序数据库，性能强
- Pull 模型（设备主动暴露数据）
- PromQL 强大
- Grafana 图表漂亮

**部署**（用 docker 一键起）：

```yaml
# docker-compose.yml
version: '3'
services:
  prometheus:
    image: prom/prometheus
    ports:
      - 9090:9090
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      
  grafana:
    image: grafana/grafana
    ports:
      - 3000:3000
```

**网络设备怎么接**：
- 用 **snmp_exporter**（SNMP 抓数据）
- 或用 **telegraf**（agent 模式）

#### LibreNMS（网络监控专用）

**官网**：https://www.librenms.org

**优势**：
- **自动发现**特别强（snmp 扫一遍就出来）
- 专门为网络设备设计
- 告警 / 拓扑 / 大屏 / 报表 全有
- 比 Zabbix 轻量

**实战**：
```bash
# 装好之后
lnms user:add admin
# 加设备
lnms device:add 192.168.1.1 v2c public
# 自动发现
lnms discover:discovery.php
# 触发报警
lnms alert:list
```

#### Node-exporter（服务器监控）

**官网**：https://github.com/prometheus/node_exporter

**干嘛的**：服务器 CPU/内存/磁盘/网络指标，配合 Prometheus 用

### 4.3 选型建议

## 第 10 部分：10 个反直觉但巨好用的命令

> 这些是"非主流"，但**关键时刻真能救命**。

### 1. `traceroute -T`（用 TCP 探测，绕过 ICMP 防火墙）

```bash
traceroute -T -p 80 8.8.8.8
# 用 TCP SYN 探测，不走 ICMP，ICMP 被墙的场合用
```

### 2. `ss -tlnp`（替代 netstat，看本地监听）

```bash
ss -tlnp
# -t: TCP
# -l: 监听中
# -n: 不解析名字
# -p: 显示进程
# 看清清楚楚哪些端口在听
```

### 3. `dig +short +trace`（看 DNS 解析全链路）

```bash
dig +trace google.com
# 一路追到根 DNS，能看出解析哪一步出问题
```

### 4. `curl -w`（看 HTTP 全程耗时）

```bash
curl -o /dev/null -s -w "DNS: %{time_namelookup}s\nConnect: %{time_connect}s\nSSL: %{time_appconnect}s\nTotal: %{time_total}s\n" https://www.baidu.com
# 一眼看哪一步慢
```

### 5. `ip -s link`（看网卡收发包/错包）

```bash
ip -s link show eth0
# 看 RX/TX 的 errors / dropped / overruns
# 比 ifconfig 详细
```

### 6. `ss -s`（看 socket 总数）

```bash
ss -s
# 看 TCP/UDP/RAW 各多少连接
# 一眼判断是不是 socket 耗尽
```

### 7. `last -F`（看完整登录日志）

```bash
last -F
# 看谁什么时候从哪登录
# 调查安全事件用
```

### 8. `lsof -i :80`（看谁在用 80 端口）

```bash
lsof -i :80
# 端口被占用了？谁占的？一目了然
```

### 9. `ethtool eth0`（看网卡硬件状态）

```bash
ethtool eth0
# Speed/Duplex/Auto-negotiation
# 网卡协商成 100M 了吗？还是 1G？
```

### 10. `watch -n 1 'command'`（每秒执行一次）

```bash
watch -n 1 'ss -tan | grep ESTAB | wc -l'
# 实时看 ESTAB 连接数变化
# 排查攻击时巨好用
```

### Bonus: 抓 HTTP 包但不解密

```bash
tcpdump -i eth0 -A -s 0 'tcp port 80 and (((ip[2:2] - ((ip[0]&0xf)<<2)) - ((tcp[12]&0xf0)>>2)) != 0)'
# 看 HTTP 明文（前提是没 HTTPS）
```
