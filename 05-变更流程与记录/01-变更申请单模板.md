### 变更实施泳道

```mermaid
sequenceDiagram
    participant Ops as 实施人
    participant Backup as 备份系统
    participant Device as 目标设备
    participant Verify as 验证
    participant Rollback as 回滚

    Ops->>Backup: 1. 备份配置
    Backup-->>Ops: 备份完成
    Ops->>Device: 2. 应用变更
    Device-->>Ops: 配置应用
    Ops->>Verify: 3. 验证业务
    Verify-->>Ops: 验证结果

    alt 验证通过
        Ops->>Device: 4. 观察 30 分钟
        Device-->>Ops: 稳定
        Ops->>Ops: 5. 关闭变更
    else 验证失败
        Ops->>Rollback: 4. 启动回滚
        Rollback->>Device: undo / reload
        Device-->>Ops: 恢复
        Ops->>Ops: 5. 复盘
    end
```

## 8. 回滚方案

> 任何一步发现异常，立即回滚！

### 8.1 一键回滚（首选）

```
# 如果是单条命令错
undo <命令>

# 如果是大段错乱
copy startup-config running-config
# 或
reload
```

### 8.2 灌入备份回滚

```
# 1. 找到变更前的备份
ls -lh 04-配置备份/历史备份/<设备名>/<变更前日期>.cfg

# 2. 灌入
scp <本地备份> admin@<设备IP>:/tmp/
ssh admin@<设备IP>
copy tftp <备份文件>
# 或
load <本地文件>
```
