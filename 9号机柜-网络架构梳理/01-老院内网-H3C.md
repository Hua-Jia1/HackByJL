### 2.1 核心交换机 1 (YKZZY-HEXIN-SW-1)

| 角色 | 主核心 (VRRP Master VLAN10/20) |
| 互联 | G1/0/24 → F1000-AK155 G1/0/0 (Vlan2) |

**VLAN 接口**：

| VLAN | IP | VRRP 状态 |
|------|----|-----------|
| 2 | 192.168.2.1/24 | → 防火墙 G1/0/0 |
| 10 | 192.168.10.1/24 | 行政办公 - VRID 1 优先级 110（主） |
| 20 | 192.168.20.1/24 | 医技设备 - VRID 2 优先级 110（主） |
| 30 | 192.168.30.1/24 | 所有科室 - VRID 3 优先级 100（备） |
| 4093 | 192.168.1.2/24 | 管理 |
| 4094 | 192.168.100.1/24 | 互连 |

**关键接口**：
- `Bridge-Aggregation1` (AGG1) - 包含 G1/0/1 和 G1/0/2，Trunk vlan 10/20/30/4094，连接核心交换机 2
- `G1/0/24` - Access vlan 2，连接防火墙 G1/0/0
- `G2/0/1` - Trunk vlan 10/20/30/4094，连接汇聚交换机

**VRRP 虚拟 IP**（与核心2 协同）：
- VLAN 10 行政办公 → 192.168.10.254（核心1 主）
- VLAN 20 医技设备 → 192.168.20.254（核心1 主）
- VLAN 30 所有科室 → 192.168.30.254（核心2 主）

**ACL 3000**（VLAN 10 入向/出向）：
- 拒绝 VLAN 20 → VLAN 10
- 拒绝 VLAN 30 → VLAN 10
- 允许其他

**ACL 3001**（G0/0/1 端口镜像 - 实际为包过滤）：
- 允许 172.31.0.0/16 ↔ 192.168.0.0/16

**静态路由**：
```
ip route-static 0.0.0.0 0 192.168.2.2          # 默认路由 → 防火墙
ip route-static 10.119.0.1 32 192.168.1.1      # → 内网网关
ip route-static 10.120.147.4 30 192.168.1.1    # → 互联网段
ip route-static 172.31.0.0 16 192.168.30.245   # → 特殊网段
```

**STP**：
- region-name: YKZZY
- instance 1 (vlan 10, 20): root primary
- instance 2 (vlan 30): root secondary

## 三、VRRP 双机热备

| VRID | VLAN | 虚拟 IP | 核心1 优先级 | 核心2 优先级 | 实际 Master |
|------|------|---------|--------------|--------------|-------------|
| 1 | 10 (行政办公) | 192.168.10.254 | 110 (主) | 100 (默认) | 核心1 |
| 2 | 20 (医技设备) | 192.168.20.254 | 110 (主) | 100 (默认) | 核心1 |
| 3 | 30 (所有科室) | 192.168.30.254 | 100 (默认) | 110 (主) | 核心2 |

**负载分担**：
- VLAN 10/20 的网关流量由核心交换机 1 处理
- VLAN 30 的网关流量由核心交换机 2 处理
- 一台核心故障时，另一台接管所有 VRRP

## 四、ACL 策略

**ACL 3000（VLAN 10 入向+出向，核心1/2 都应用）**：
```
rule 5  deny   ip source 192.168.20.0 0.0.0.255 destination 192.168.10.0 0.0.0.255
rule 10 deny   ip source 192.168.30.0 0.0.0.255 destination 192.168.10.0 0.0.0.255
rule 15 permit ip
```
> 效果：禁止 医技设备(V20) 和 所有科室(V30) 访问 行政办公(V10)，但允许反向及其他通信

**ACL 3001（G0/0/1 包过滤，核心1）**：
```
rule 1 permit ip source 172.31.0.0 0.0.255.255 destination 192.168.0.0 0.0.255.255
rule 2 permit ip source 192.168.0.0 0.0.255.255 destination 172.31.0.0 0.0.255.255
```
> 效果：允许 172.31.0.0/16（内网网关互联段）和 192.168.0.0/16（医院内网）双向通信

**ACL 3005（防火墙 SSH）**：
```
rule 0 permit ip source 192.168.10.111 0
rule 5 deny ip
```
> 效果：仅允许 192.168.10.111 SSH 登录

## 五、冗余/可靠性

- **核心层**：核心1+核心2 通过 AGG1 链路聚合互联（2×G1/0/1-2）
- **核心层 ↔ 防火墙**：核心1↔FW G1/0/0，核心2↔FW G1/0/1（双归属）
- **核心层 ↔ 汇聚**：G2/0/1 单链路上联（无冗余）
- **VRRP**：三层网关双机热备，支持负载分担
- **STP**：MSTP 多实例（YKZZY region），instance 1 (V10/20) 核心1 root，instance 2 (V30) 核心2 root
