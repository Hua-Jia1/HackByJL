# 安全策略与 ACL

## 一、防火墙安全策略（F1000-AK155）

### 1.1 安全域定义

| 安全域 | 接口 | 用途 |
|--------|------|------|
| **Local** | 本机 | 设备自身 |
| **Trust** | G1/0/0, G1/0/1, G1/0/5, G1/0/15 | 内部受信任网络（核心、内网网关） |
| **DMZ** | G1/0/2, G1/0/2.40, Vlan40 | 服务器区 |
| **Untrust** | G1/0/3, G1/0/4 | 互联网（NAT）、外联 |
| **Management** | - | 管理（未使用） |

### 1.2 域间安全策略

| 序号 | 规则名 | 源域 | 目的域 | 源地址 | 目的地址 | 服务 | 动作 |
|------|--------|------|--------|--------|----------|------|------|
| 0 | trust-dmz | Trust | DMZ | Client (V10/20/30/200-203/205) | 服务器区 (V40, V192/20) | C-S (开放端口) | pass + 日志 + 计数 |

| 2 | local-dmz | Local | DMZ | - | - | any | pass + 日志 + 计数 |
| 3 | trust-local | Trust | Local | - | - | any | pass + 日志 + 计数 |
| 4 | local-trust | Local | Trust | - | - | any | pass + 日志 + 计数 |
| 5 | dmz-local | DMZ | Local | - | - | any | pass + 日志 + 计数 |
| 7 | untrust_local | Untrust | Local | - | - | any | pass + 计数 |
| 8 | local_untrust | Local | Untrust | - | - | any | pass + 计数 |
| 9 | dmz_untrust | DMZ | Untrust | 服务器区 | XieTongMen-Client | any | pass + 计数 |
| 10 | untrust_dmz | Untrust | DMZ | XieTongMen-Client | 服务器区 | C-S | pass + 计数 |

**特点**：
- Trust → DMZ 仅允许 C-S 服务（精细化）

- 启用日志和计数便于审计

## 二、核心交换机 ACL 3000（VLAN 10 互访限制）

**核心1 和 核心2 都应用了 ACL 3000** 到 Vlan-interface10（双向）：

```
rule 5  deny   ip source 192.168.20.0 0.0.0.255 destination 192.168.10.0 0.0.0.255
rule 10 deny   ip source 192.168.30.0 0.0.0.255 destination 192.168.10.0 0.0.0.255
rule 15 permit ip
```

**效果**：
- ❌ VLAN 20 医技设备 → VLAN 10 行政办公：禁止
- ❌ VLAN 30 所有科室 → VLAN 10 行政办公：禁止
- ✅ 其他通信：允许

> **行政办公区（V10）单向隔离**——保护办公网络安全，防止业务/科室网络访问办公区

## 三、核心交换机 ACL 3001（G0/0/1 端口过滤）

**仅核心1 配置**（应用于 G0/0/1）：

```
rule 1 permit ip source 172.31.0.0 0.0.255.255 destination 192.168.0.0 0.0.255.255
rule 2 permit ip source 192.168.0.0 0.0.255.255 destination 172.31.0.0 0.0.255.255
```

> 允许 172.31.0.0/16（内网网关互联段）与 192.168.0.0/16（医院内网）双向通信

## 四、SSH 管理（核心交换机 ACL 3005）

**核心1 和 核心2 都配置了 ACL 3005**：

```
rule 10 deny ip
```

## 五、N-core IP+MAC 绑定

N-core 配置了大量 `address-bind`（超过 200 条），主要分布在：
- `192.168.200.x`：业务终端
- `192.168.201.x`：业务终端（最密集）
- `192.168.202.x`：业务终端
- `192.168.203.x`：业务终端
- `192.168.255.x`：管理/服务器

> 全部绑定的 uplink 为 `AggregatePort 8`（即上联 RG-EG3220 的链路）
> **作用**：防止 ARP 欺骗、IP 盗用，提升新院网络安全性

## 六、NAT 策略

### 6.1 老院防火墙 NAT
- G1/0/3 (10.120.147.6/30) 配置了 `nat outbound`
- 内网用户通过防火墙访问互联网

### 6.2 内网网关 RG-EG3220 NAT
```
ip nat inside source list 1 pool nat_pool overload
```
- pool: Gi0/5 接口地址 (172.31.254.1)
- 内网用户经老院防火墙二次 NAT 上网

### 6.3 外网网关 RG-EG3230 NAT
```

 address interface Gi0/7 match interface Gi0/7
ip nat inside source list 1 pool nat_pool overload
```
- pool: Gi0/7 接口地址 (10.2.2.2)
- 外网用户通过 Gi0/7 (WAN) 出公网

## 七、AAA 认证

### 7.1 核心防火墙 F1000-AK155

```
local-user zkyc class manage
 password hash $h$6$...
 service-type ssh terminal http https
 authorization-attribute user-role level-3
 authorization-attribute user-role network-admin
 authorization-attribute user-role network-operator
 password-control login-attempt 5 exceed lock-time 10
```

- 用户名：`zkyc`
- 服务：SSH/Terminal/HTTP/HTTPS
- 角色：level-3、network-admin、network-operator
- 密码策略：5 次失败锁定 10 分钟

### 7.2 H3C 核心交换机

```
local-user zkyc class manage
 password hash $h$6$...
 service-type ssh terminal
 authorization-attribute user-role network-admin
 authorization-attribute user-role network-operator
 password-control login-attempt 5 exceed lock-time 10
```

- 同上
- 服务：SSH/Terminal（无 HTTP）

### 7.3 N-core / RG-EG3220 等 Ruijie 设备

```
username admin password 7 $10$1a7$3rsQYUPXJ8Szfg==$
enable secret 8 ...
enable password 7 ...
password policy printable-character-check
password policy min-size 8
password policy strong
```

- 用户名：`admin`
- 密码：加密存储
- 强密码策略（8位+，含特殊字符）
- SSH/Telnet 启用

### 7.4 AC WS6008
