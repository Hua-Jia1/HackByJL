# 无线网络与 DHCP 服务

## 一、无线网络架构

```
                              ┌────────────────────────────┐
                              │  AC WS6008 (U35)           │
                              │  192.168.200.3/24          │
                              │  SSID: Zangyiyuan          │
                              │  PSK: XXK123123            │
                              │  tunnel local              │
                              └─────┬──────────────────────┘
                                    │ CAPWAP (capwap ctrl-ip 192.168.200.3)
                                    │
              ┌─────────────────────┴─────────────────────┐
              │                                           │
              ▼                                           ▼
    ┌──────────────────────┐               ┌──────────────────────┐
    │  AP (Vlan254 管理)   │               │  AP (Vlan254 管理)   │
    │  192.168.254.x       │               │  192.168.254.x       │
    │  DHCP option138=AC   │               │  DHCP option138=AC   │
    └──────────┬───────────┘               └──────────┬───────────┘
               │ 无线接入                            │ 无线接入
               ▼                                      ▼
    ┌──────────────────────┐               ┌──────────────────────┐
    │ 无线终端 (STA)       │               │ 无线终端 (STA)       │
    │ 192.168.236.x/22     │               │ 192.168.240-250.x    │
    │ 或 192.168.240-250.x │               │                      │
    └──────────────────────┘               └──────────────────────┘
```

## 二、SSID 与认证

| 项目 | 配置 |
|------|------|
| **SSID** | `Zangyiyuan` (藏医院) |
| **WLAN ID** | 1 |
| **加密方式** | WPA2-PSK / WPA-PSK |
| **PSK 密码** | `XXK123123` |
| **AKM** | PSK (预共享密钥) |
| **加密套件** | AES |
| **隧道模式** | `tunnel local`（本地转发） |

> 注意：RG-EG3230 同时也配置了 `wlan-config 1 Zangyiyang`（少一个字母），说明 EG 本身也作为 AP 控制器可独立管理无线（CAPWAP 源 IP 3.3.33.3）。

## 三、AP 管理

| 项目 | 配置 |
|------|------|
| AP 管理 VLAN | Vlan254 |
| AP 管理网段 | 192.168.254.0/24 |
| AP 网关 | 192.168.254.1 (W-core) |
| AC 地址（DHCP option 138）| 192.168.200.3 |

**DHCP option 138 流程**：
1. AP 上电 → DHCP 请求（Vlan254）
2. W-core `ap_pool` 提供 IP + option 138 = 192.168.200.3
3. AP 通过 CAPWAP 协议向 AC 注册
4. AC 推送 SSID、加密、隧道等配置

## 四、STA 终端 VLAN

| VLAN | 网段 | 用途 |
|------|------|------|
| 236 | 192.168.236.0/22 | STA 终端超网（无线终端聚合） |
| 240 | 192.168.240.0/24 | 无线业务 |
| 241-250 | 192.168.24x.0/24 | 无线业务（细分） |

**STA 池**（sta_pool）：192.168.236.0/22（覆盖 192.168.236.0 - 192.168.239.255）

> 无线终端先获取 Vlan236 的 IP（/22 超网），再根据策略分配到具体 Vlan240-250

## 六、CAPWAP 隧道

| 设备 | CAPWAP 源 IP | AC 地址 | 模式 |
|------|--------------|---------|------|
| RG-EG3230 (U37) | 3.3.33.3 (Loopback 0) | - | `no ac-control disable`（启用 AP 控制） |
| WS6008 (U35) | 192.168.200.3 | - | `capwap ctrl-ip 192.168.200.3` |
| 无线 AP | (DHCP 分配) | 192.168.200.3 | - |

> **两套 AP 控制系统并存**：
> 1. **WS6008** 独立 AC，管理一部分 AP（CAPWAP 集中/本地转发）
> 2. **RG-EG3230** 内置 AC 功能（capwap 模式），可旁挂管理一部分 AP
