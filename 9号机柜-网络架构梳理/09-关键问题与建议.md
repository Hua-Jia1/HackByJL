# 关键问题与运维建议

> 通过配置分析发现的问题、优化建议

## 一、配置中发现的问题

### 1.1 安全风险

| 风险 | 位置 | 描述 | 建议 |
|------|------|------|------|
| 单一管理 IP | F1000-AK155 / H3C 核心 | SSH 仅允许 192.168.10.111 单 IP | 增加冗余管理 IP，限制子网而非单 IP |
| ACL 3005 单点 | 防火墙/核心 | 单点失败将导致无法远程管理 | 配置带外管理或 console 备用 |
| N-core SNMP 关闭 | N-core / W-core | `no enable service snmp-agent` | 启用 SNMPv3 用于监控告警 |
| 无线 PSK 单一 | WS6008 | 全网使用同一 PSK `XXK123123` | 考虑 802.1X + RADIUS 认证，按用户授权 |
| 默认登录密码 | 部分配置 | `password ruijie@123` 明文/弱密码 | 全部改为强密码 + 定期轮换 |
| 锐捷 RG-EG3230 WAN 暴露 Telnet | RG-EG3230 | `enable service telnet-server` 可能默认开启 | 确认 Telnet 已禁用，仅 SSH |
| 跨网段 IP-MAC 绑定 | N-core | 单个 MAC 绑到 192.168.200.x 多个 IP | 同一 MAC 多次出现，可能是冗余设计，需核实 |

### 1.2 配置不一致

| 不一致 | 描述 |
|--------|------|
| WS6008 启用 CAPWAP 但 RG-EG3230 也启用 | 两套 AC 可能冲突，需要明确主备 |
| 老院 N-core 的 vlan10/20/30/40 DHCP | N-core 也提供老院 VLAN 的 DHCP，可能与 H3C 核心冲突 |
| `N-EG-RG220` 主机名 | 实际型号是 EG3220，主机名带 RG220，可能造成混淆 |
| N-core 主机名重复 | `#1` 和 `#2` 都叫 `N-CORE`，仅靠 VSU 区分 |
| 老院/新院 Vlan255 重叠 | 老院 H3C 用 4094 管理，新院 Ruijie 用 255 管理，避免冲突 |

### 1.3 网络设计

| 设计点 | 描述 |
|--------|------|
| 192.168.0.0/16 大网段 | 整个医院用一个 /16 汇总路由，可能影响精细化路由控制 |
| RG-EG3230 路由回环 | `ip route 192.168.0.0/16 → W-core` + W-core 默认 → RG-EG3230 |
| W-core 默认路由到 W-EG | W-core 是二层设备但默认路由指向 W-EG，依赖 W-EG 处理路由 |
| 服务器镜像端口分散 | 服务器交换机镜像了部分端口而非全部，监控覆盖不全 |

## 二、运维建议

### 2.1 监控

- **日志中心**：所有设备 `info-center loghost` 指向 192.168.40.16，建议部署日志服务器统一收集
- **SNMP 监控**：启用 SNMPv3，对接 Zabbix/Prometheus + Grafana
- **流量监控**：在核心 N-core / W-core / F1000-AK155 配置 sFlow/NetStream 上送分析
- **无线空口质量**：WS6008 配置 AP 性能上报

### 2.2 备份

- 所有设备配置定期备份（每周）
- 使用脚本 `show running-config` + TFTP/SCP 备份
- 配置变更前后对比

### 2.3 高可用

| 系统 | 现状 | 建议 |
|------|------|------|
| 老院核心层 | VRRP + 链路聚合 + MSTP | ✓ 良好 |
| 老院防火墙 | 单台 F1000-AK155 | ⚠️ 建议增加 HA 双机 |
| 老院服务器交换机 | 单台 S5120V2 | ⚠️ 建议堆叠或双机 |
| 新院核心层 | VSU 堆叠 (2 台 RG-S5760C) | ✓ 良好 |
| 内网网关 | 单台 RG-EG3220 | ⚠️ 建议双机或换为集群 |
| 外网网关 | 单台 RG-EG3230 | ⚠️ 建议双机或换为集群 |
| 外网核心 | 单台 W-core | ⚠️ 建议增加冗余或冷备 |
| AC 控制器 | 单台 WS6008 | ⚠️ 建议主备或 2+1 集群 |

### 2.4 VLAN 规划扩展

| 现有 | 建议 |
|------|------|
| VLAN 200-205 业务 | 扩展到 200-212 已预留 |
| 无线 240-250 | 充足，无需扩展 |
| 设备管理 255 | 充足（/25 可容纳 126 设备） |
| 服务器 40 | 老院已使用，建议同步扩展新院 |

## 三、备份与恢复

### 3.1 配置备份命令

**Ruijie 设备**：
```
copy running-config tftp://<server>/<filename>
copy startup-config tftp://<server>/<filename>
```

**H3C 设备**：
```
save    # 保存到 startup
backup startup-configuration to <server><filename>
```

### 3.2 关键配置归档

建议保存：
- N-core 完整配置（`show running-config`）
- F1000-AK155 完整配置
- H3C 核心 1/2 完整配置
- RG-EG3220/3230 完整配置
- W-core 完整配置
- WS6008 完整配置

## 四、扩展建议

### 4.1 业务扩展

| 业务 | VLAN 建议 |
|------|----------|
| 5G 专网 | VLAN 207-209 |
| IoT 医疗设备 | VLAN 210-212 |
| 视频会议 | VLAN 252 |
| 互联网+医疗（外联） | VLAN 60-70 已有 |

### 4.2 安全加固

1. 启用 RADIUS/TACACS+ 集中认证
2. 部署态势感知平台
3. 启用端口安全（port-security）
4. DHCP Snooping 全网部署
5. DAI（Dynamic ARP Inspection）
6. 部署 VPN 接入（SSL VPN 已有 RG-EG 平台）
