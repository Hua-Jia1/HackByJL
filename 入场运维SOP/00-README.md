## 第一周行动清单

> 打印出来，挨个打勾

###  入厂
- [ ] 拿到"四件套"：资产清单 / 拓扑图 / IP 规划 / 账号
- [ ] 申请所有需要的账号
- [ ] 装好 MobaXterm + VS Code + Bitwarden
- [ ] 拿到 Console 线，找个空闲 PC 测通
- [ ] 走一遍机房，认识物理位置
- [ ] 拍机房照片，标注机柜 U 位

###  信息采集
- [ ] 编辑 `03-自动化脚本/01-批量采集设备配置.py`，填设备信息
- [ ] 跑批量采集脚本
- [ ] 看采集结果：
  - [ ] running-config 和 startup-config 一致？
  - [ ] 接口有错包？
  - [ ] 时钟同步？
  - [ ] 路由邻居稳？
  - [ ] CPU/内存正常？
- [ ] 把异常登记到 `05-变更流程与记录/02-入厂隐患清单.md`

### 拓扑与备份
- [ ] 照着实际配置画拓扑图
- [ ] 和文档拓扑对比，差异点列出来
- [ ] 部署 Oxidized 或跑通自动备份脚本
- [ ] 验证备份：随便找台设备，把备份灌进旧设备

###  监控与文档
- [ ] 把所有设备加入监控（Zabbix / LibreNMS）
- [ ] 设置 NTP，所有设备同步
- [ ] 设置 syslog 集中收集
- [ ] 写完资产清单 / IP 规划表
- [ ] 写完"如果我明天请假一个月"文档

## 几条铁律

> 1. **没有备份不动配置**
> 2. **没有方案不动配置**
> 3. **核心设备改配置必须有第二人在场**
> 4. **变更在窗口期做**
> 5. **改完配置观察 30 分钟再走**
> 6. **一切操作留痕**
> 7. **不懂就问，别猜**
