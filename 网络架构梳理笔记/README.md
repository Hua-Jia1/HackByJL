### 4 块之间的"大门"

| 从 → 到 | 通过什么设备 | 用途 |
|---------|-------------|------|
| 办公网 → 互联网 | 锐捷 N-core → **EG3220** → 防火墙 F1000 → MSR3620 | 办公上网 |
| 医疗网 → 服务器 | H3C 核心 → **F1000** Trust→DMZ | 医生看 PACS |
| 无线 → 互联网 | 锐捷 W-core → **EG3230** → ISP | 访客 WiFi |
| 远程 → 内网 | 外网 → **华为 USG** → 内部 | 远程办公 |

### 5 个核心概念(必须懂)

| 概念 | 一句话 | 例子 |
|------|--------|------|
| **VLAN** | 把一台交换机切成多块 | VLAN 10 = 行政办公(192.168.10.x) |
| **VRRP** | 两台核心"假装"是一台 | 192.168.10.254 = SW1/SW2 共享虚拟 IP |
| **VSU 堆叠** | 多台物理机"假装"是 1 台 | N-core #1 + #2 实际是一个逻辑设备 |
| **聚合 (LACP)** | 多根网线绑成 1 根 | SW1+SW2 用 Bridge-Agg1 互联 |
| **Trust/DMZ/Untrust** | 防火墙把网络分 3 个安全级别 | 内网=Trust, 服务器=DMZ, 互联网=Untrust |

### 3 个业务一句话

| 业务 | 流量路径 |
|------|---------|
| 行政办公上网 | PC → N-core → EG3220(NAT)→ F1000 → MSR3620 → 互联网 |
| 医生看 PACS | PC → H3C 核心 → F1000(Trust→DMZ)→ S5120V2 → 服务器 |
| 访客连 WiFi | 手机 → AP → AC → W-core → EG3230 → ISP |

### 9 个乱点(速记版)

1. 4 个"核心"挤一起,边界不清
2. VLAN 10 在 H3C 和 N-core 各出现一次,IP 一样
3. N-core #1/#2 疑为同一 VSU 堆叠,不是真双机
4. W-core 有 2 份配置,**不知道哪份生效**
5. 命名 N/W/HJ 含义混搭
6. 至少 3 套 DHCP 在抢 VLAN 10/20
7. MSR3620 路由器不在本机柜但被防火墙引用
8. 华为 USG6000E/S5735S 没配置文档
9. VLAN 4 (10.12.4.0/24) 用途不明
